# pweb-parking-discount-helper

> pweb.kr 주차 할인등록 화면에서 **"0원까지 할인권을 몇 장 더 넣어야 하는지"**를 계산해 주는 북마클릿
> (북마크명 "할인계산" · 정식 명칭 "pweb 주차할인 계산기")

## 이게 뭔가요

`pweb.kr` 기반 주차 할인등록 화면은 차량을 조회하면 입차시각과 경과시간만 보여줍니다.
정작 필요한 **"지금 주차권을 얼마나 더 적용해야 정산이 0원으로 끝나는지"**는 알려주지 않습니다.
예전 건물 자체 시스템엔 있던 값인데 사라져서, 매번 암산해야 했습니다.

이 도구는 그 숫자를 되살립니다. 그게 전부입니다.

## 무엇을 해주나

- **부족분 계산 + 최소비용 조합 추천** — 아직 0원이 아니면, 얼마가 부족한지와 그걸 가장 싸게 채우는 할인권 조합을 추천합니다. 딱 맞추지 않고 항상 조금 넘겨 안전하게 덮습니다.
- **0원 상태 실시간 카운트다운** — 이미 0원이면 "앞으로 얼마나 여유가 있는지 / 몇 시까지 0원인지"를 1초마다 갱신해 보여줍니다.
- **모바일 지원** — 폰에서는 화면 하단 시트로 뜨고 글씨를 키워 보정합니다(실제 주 사용처가 폰).

## 왜 북마클릿인가

할인등록 데이터는 로그인해야 보이므로 다른 도메인의 웹페이지에서는 읽을 수 없습니다(CORS·쿠키 미공유).
데이터를 읽을 수 있는 곳은 로그인된 그 페이지 안뿐입니다. 그래서 북마클릿입니다.

- 이미 로그인해 열어둔 페이지 위에서, 사용자가 누를 때만 동작합니다.
- 페이지가 이미 불러온 값만 읽습니다. **외부 전송·저장·자동 로그인 없음.**
- **주차권을 실제로 적용하지 않습니다.** 운영 중인 실제 시스템이라 잔여량에서 돈이 빠지므로, 오로지 화면의 값을 읽어 계산만 합니다.

## 어느 건물이든 동작

주소 앞부분(예: `a23021`)은 건물 번호일 뿐, 도메인을 하드코딩하지 않습니다.
`pweb.kr` 할인등록 화면을 쓰는 곳이면 어느 건물에서든 동작합니다(전국 수천 개 현장이 같은 플랫폼).
건물마다 구조가 달라 계산이 어긋날 수 있는 경우엔, 스스로 감지해 멈추고 "사용 금지"를 표시합니다 — 틀린 숫자보다 아무것도 안 보여주는 편이 낫기 때문입니다.

## 구성

| 파일 | 설명 |
|---|---|
| `install.html` | 설치·사용 안내 페이지. 북마클릿을 즐겨찾기로 끌어다 놓거나, 폰에서 코드를 복사해 등록. 원본 스크립트를 런타임에 `javascript:` URL로 조립합니다. |
| `bookmarklet-source.js` | 같은 스크립트의 주석 달린 읽기용 원본. **두 파일의 로직은 항상 같아야 합니다 — 고칠 땐 둘 다.** |

## 설치

`install.html`을 브라우저로 열어 안내를 따르면 됩니다(PC는 즐겨찾기 바로 드래그, 폰은 코드 복사 후 북마크 URL 교체).
