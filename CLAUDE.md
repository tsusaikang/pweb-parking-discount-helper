# CLAUDE.md

pweb.kr 웹 할인등록 사이트 보조 도구 (북마클릿 "pweb 주차할인 계산기",
주소창 입력용 짧은 북마크명 "할인계산"). 회사(광명 GIDC)는 티맵주차로 바뀐
`https://a23021.pweb.kr` 을 쓰지만 **주소 앞부분(a23021)은 건물 번호일 뿐이다** —
도메인을 하드코딩하지 않으므로 어느 건물이든 동작한다 (2026-07-16 사용자 확인).
**pweb.kr(주차관제 벤더의 웹 할인관리시스템)이 제공하는 사이트라 직접 수정할 수 없다.**
그래서 래핑한다.

이름은 원래 "주차 0원 도우미/주차0원" 이었으나 2026-07-21 변경했다. 이유:
(1) 이 시스템은 티맵 전용이 아니라 pweb.kr 플랫폼이며 전국 약 2,750개 현장이
같은 구조를 쓴다(www.pweb.kr → 사용자 리스트에서 확인). (2) "주차0원" 은
요금을 눈속임으로 없애는 트릭처럼 들려, 실제 기능(정식 할인권을 몇 장 적용해야
정산이 끝나는지 계산)과 맞지 않았다. (3) pweb 전용이라는 호환 범위를 제목에
반영했다 — compat() 가드 철학과 일치. 사람이 매일 누르는 북마크엔 pweb 없이
"할인계산", 설명이 필요한 제목·안내문에만 "pweb 주차할인 계산기".

## 해결하려는 불편

차량번호 조회 시 **입차시각과 경과시간만** 나오고, 주차권 적용 후
**"얼마나 더 적용해야 0원이 되는지"** 가 없다. 예전 건물 자체 시스템엔 있었다.
사용자가 매번 암산하고 있었다. 그 값을 계산해 보여주는 게 이 도구의 전부다.

## 산출물

- `install.html` — 설치·사용 안내 + 북마클릿을 즐겨찾기로 끌어다 놓는 페이지.
  `<script type="text/plain" id="src">` 안의 원본을 읽어 `javascript:`+encodeURIComponent 로
  href 를 **런타임에 만든다**. 손으로 minify 하지 말 것.
- `bookmarklet-source.js` — 같은 스크립트의 주석 달린 읽기용 원본.
  **두 파일의 로직은 같아야 한다. 고칠 땐 반드시 둘 다.**

## 형태를 북마클릿으로 정한 이유 (2026-07-16 확정)

처음엔 별도 웹페이지(Cloudflare 배포)를 검토했으나 **불가능하다.**
로그인이 필요한 데이터라 다른 도메인에서는 CORS 와 쿠키 미공유로 API 를 못 읽는다.
데이터를 읽을 수 있는 곳은 `a23021.pweb.kr` 페이지 안뿐이다.

사용자 방침: **"새 도구를 사용하지 않은 상태에서 자동 로그인 되는 딱 그 수준만 유지."**
더 넓은 자동화(계정 저장·자동 로그인)는 개인정보 이슈라 하지 않는다.
북마클릿은 이 방침과 정확히 맞는다 — 이미 로그인된 페이지에서 사용자가 누를 때만
돌고, 페이지가 이미 불러온 값만 읽고, 외부 전송·저장이 없다. **이 성질을 깨지 말 것.**

Tampermonkey 유저스크립트는 회사 PC 에 설치가 어려울 수 있어 택하지 않았다.

## 공개 배포·버전 알림 (2026-07-21 사용자 요청)

install.html 을 공개 도메인(가칭 `pweb-parking-help.kr` — **미확정**, 확정 시
두 파일의 `HOME` 상수와 version.json 의 `page` 를 고칠 것)에 올려 배포한다.
북마클릿은 자동 업데이트가 불가능하므로: 패널을 열 때 1회 `HOME/version.json` 을
fetch 해 `VERSION` 상수와 다르면 패널 상단에 깜빡이는(👉 `__pk_blink` keyframes)
주황 배너 + 새창 링크를 띄운다. **데이터는 아무것도 보내지 않는 정적 파일 GET
하나뿐** — "외부 전송 없음" 방침의 유일한 예외로 사용자가 승인함. 실패는 무시
(계산 기능과 무관). 배포 절차: 두 파일 VERSION ↑ → version.json 같은 값으로 ↑
→ 재배포. `_headers` 는 Cloudflare Pages 용 CORS 설정(GitHub Pages 는 불필요).

## 절대 규칙

- **주차권을 실제로 적용하지 말 것** (2026-07-16 사용자 지시). 운영 중인 실제
  주차 시스템이고 잔여량에서 돈이 빠진다. 조사·검증은 **이미 할인이 적용된 차량의
  표시를 읽는 것**으로 갈음한다 (그렇게 해서 이번 조사를 다 끝냈다).
  부득이 적용해야 하면 무료2시간·30분권 1장까지만, 확인 즉시 **반드시 적용 취소**.
- 조회는 읽기라 괜찮다.

## 사이트 구조 (2026-07-16 조사)

할인등록 화면: `/discount/registration`

**API** (페이지가 내부적으로 호출. 직접 부를 일은 없다)

- `POST /discount/registration/listForDiscount` — 차량 목록 조회 (`fncDoListMst()`)
- `POST /discount/registration/getForDiscount` — 선택 차량 상세 (`fncDetailInfo(iID, iCardType)`)

**도구가 읽는 전역 (이게 전부다. API 를 직접 부르지 않는다)**

| 소스 | 내용 |
|---|---|
| `window.dataSetMst` | 조회된 차량 목록. `id`·`carNo`·`incar_min`(경과분)·`dscnt_cnt` |
| `#peId` (input) | 현재 선택된 차량 id → `dataSetMst` 에서 찾아 쓴다 |
| `window.dataSetDtl` | 선택 차량에 **이미 적용된 할인** 목록. `discount_name`·`dc_time`(차감분)·`discountTypeId` |
| `#differentTime` (span) | 주차시간 사람이 읽는 형식("6시간 20분"). `dataSetMst` 없을 때 폴백 |

## 할인권 (사이트 `listDiscountType` 실측값)

전부 `discount_type:"1"` = **시간 차감식**. 요금(원)이 아니라 분을 깎는다.

| id | 이름 | 차감 | 가격 | 비고 |
|---|---|---|---|---|
| 5 | 무료2시간(평일) | 120분 | 0원 | `dup_limit:1` → 1회만 |
| 1 | 30분할인 | 30분 | 500원 | |
| 2 | 1시간할인 | 60분 | 1,000원 | |
| 3 | 4시간할인 | 240분 | 4,000원 | |
| 4 | 24시간할인 | 1,440분 | 15,000원 | 분당 단가가 유일하게 싸다 |

## 계산 규칙 (2026-07-16 사용자 확정)

```
커버 = 기본무료 30분 + Σ(적용된 할인 dc_time)
여유 = 커버 − 경과시간
  여유 >= 0  → 0원. "앞으로 N분 여유" 표시
  여유 <  0  → 부족. 무료2시간(미사용 시)을 먼저 쓰고,
               남은 만큼을 최소 비용 조합(DP)으로 채운다
```

- **기본 무료 30분**은 사용자가 확정해 준 이 주차장 규칙이다. 사이트 데이터에서
  유도한 값이 아니다. 규칙이 바뀌면 `BASE_FREE` 를 고친다.
- **안전하게 올린다** — 추천 조합은 항상 경과시간을 넘겨 덮는다. 딱 맞추지 않는다.
- 사용자가 "앞으로 얼마나 여유 있는지"도 보고 싶어했다. 0원 상태에서도 여유 분을 표시.

## 함정

- **이 관리자 화면은 요금(원)을 계산하지 않는다.** `finalFee` 는 항상 `""`,
  `parkEntry.dFee` 는 항상 `0`이다 — 20시간 주차한 무할인 차량도 그렇다.
  요금은 출차 시점에 계산되는 듯하다. **`finalFee`/`dFee` 에 기대지 마라.**
  그래서 이 도구는 전부 **시간**으로 계산한다.
- **`grace_time` 은 쓰지 마라.** 값이 -1, 181, 888, 1290, 112336 으로 들쭉날쭉하고
  `incar_min` 과 상관관계가 없다. 의미를 특정하지 못했다. 무시하기로 했다.
- **할인내역 표는 `<table>` 이 아니다.** dhtmlx 그리드(`XgridDtl`)라 DOM 에서
  `<table>` 로 긁으면 헤더만 나온다. 적용된 할인은 `window.dataSetDtl` 에서 읽어야 한다.
- **`javascript_tool` 로 조사할 때 응답 원문을 그대로 반환하면 차단된다**
  ("Cookie/query string data"). 입차이미지 URL 의 쿼리스트링 때문으로 보인다.
  구조·키만 뽑거나 숫자를 마스킹해서 반환하면 통과한다.
- **file:// 로 install.html 을 브라우저 자동화로 못 연다.** 사용자가 직접 더블클릭하면 된다.
- 북마클릿은 다시 누르면 닫히는 토글이다. 패널 갱신은 `#peId` 를 1초마다 폴링한다
  (사이트 함수를 감싸지 않는다 — 사이트 코드에 손대지 않는 게 안전하다).

## 모바일 지원 (2026-07-16 사용자 요청 — 실제 주 사용처가 폰)

- `matchMedia('(pointer:coarse)')` 로 터치 기기를 감지해 패널을 **하단 시트**로.
  사이트에 모바일 뷰포트 설정이 없어 폰에서 축소 렌더링될 수 있으므로
  `zoom`(1.3~2.8, innerWidth/400)으로 키운다. max-height 는 zoom 만큼 나눠 보정.
- 드래그 이동은 데스크톱 전용. 하단 시트는 고정 + 스크롤.
- install.html 에 Android Chrome(주소창에 북마크 이름 입력 실행) /
  iPhone Safari(즐겨찾기 URL 교체) / 삼성 인터넷 설치법 수록.
  북마크 이름은 `할인계산` 추천(주소창 입력용). 복사는 navigator.clipboard
  우선, execCommand 폴백.

## 호환성 가드 (2026-07-16 사용자 요청)

건물마다 사이트 구조가 다를 수 있다. 매 폴링마다 `compat()` 로 기대 구조를
검사하고, 하나라도 어긋나면 **폴링을 완전히 멈추고 ⛔사용금지 화면**만 남긴다
(계산 UI 전부 제거, "강주상 (tsusai@msn.com) 에게 문의" + 기술 사유 표시).
검사 항목: `#peId` 존재, `fncDoListMst`·`fncDetailInfo` 함수 존재,
`dataSetMst` 배열 + `id`/`carNo`/`incar_min` 필드 + incar_min 숫자,
`dataSetDtl` 배열 + `discountTypeId`/`dc_time`/`discount_name` 필드 + dc_time 숫자.
데이터가 아직 없으면(조회 전) 정적 검사만 통과하면 대기 상태로 둔다.

## UI 동작 (2026-07-16 개편, 사용자 확정)

- 추천 주차권은 **카드 리스트**. 패널을 연 시점의 적용내역을 기준(base)으로
  스냅샷해 두고, 이후 `dataSetDtl` 에 늘어난 할인은 **✓적용됨 카드**로 표시.
  남은 부족분은 매 폴링마다 **재계산** — 추천과 다른 권종을 적용해도 맞춰 간다
  (특정 권종이 소진돼 대체 사용하는 경우가 실제로 있음).
- base 스냅샷은 차량 선택 **1초 뒤**에 뜬다 (`getForDiscount` 가 비동기라
  `dataSetDtl` 이 늦게 도착하기 때문).
- 완료/부족/대기는 **헤더 색 + 배지**(초록 ✓ 0원 완료 / 빨강 ⚠ 적용 필요 / 회색 대기).
- 0원 상태는 "N시간 N분 남음 · **HH:MM까지** 0원" 을 크게 표시.
  `incar_min` 은 재조회 전까지 고정이므로 값이 갱신된 시각(seenAt)을 기억해
  현재 경과를 보간, 1초마다 실시간 카운트다운한다.

## 조사 방법 메모

차량번호는 **2자리 이상**이면 검색된다. 입차 중인 차를 모를 땐 흔한 조합("23" 등)을
넣으면 수십 대가 잡힌다. 그중 `dscnt_cnt > 0` 인 차를 선택하면 `dataSetDtl` 에
적용된 할인이 들어와, **주차권을 적용해 보지 않고도** 표시 방식을 확인할 수 있다.
