# CLAUDE.md

pweb.kr 웹 할인등록 사이트 보조 도구 (북마클릿 "pweb 주차할인 계산기",
주소창 입력용 짧은 북마크명 "할인계산"). 회사(광명 GIDC)는 티맵주차로 바뀐
`https://a23021.pweb.kr` 을 쓰지만 **주소 앞부분(a23021)은 건물 번호일 뿐이다** —
도메인을 하드코딩하지 않으므로 어느 건물이든 동작한다 (2026-07-16 사용자 확인).
**pweb.kr(주차관제 벤더의 웹 할인관리시스템)이 제공하는 사이트라 직접 수정할 수 없다.**
그래서 래핑한다.

이름은 원래 "주차 0원 도우미/주차0원" 이었으나 2026-07-21 변경했다. 이유:
(1) 이 시스템은 티맵 전용이 아니라 pweb.kr 플랫폼이며 전국 약 2,750개 현장이
같은 구조를 쓴다(www.pweb.kr → 사용자 리스트에서 확인). (2) "주차0원" 은
요금을 눈속임으로 없애는 트릭처럼 들려, 실제 기능(정식 할인권을 몇 장 적용해야
정산이 끝나는지 계산)과 맞지 않았다. (3) pweb 전용이라는 호환 범위를 제목에
반영했다 — compat() 가드 철학과 일치. 사람이 매일 누르는 북마크엔 pweb 없이
"할인계산", 설명이 필요한 제목·안내문에만 "pweb 주차할인 계산기".

## 해결하려는 불편

차량번호 조회 시 **입차시각과 경과시간만** 나오고, 주차권 적용 후
**"얼마나 더 적용해야 0원이 되는지"** 가 없다. 예전 건물 자체 시스템엔 있었다.
사용자가 매번 암산하고 있었다. 그 값을 계산해 보여주는 게 이 도구의 전부다.

## 로더 방식 (2026-07-21 사용자 승인)

version.json 은 이제 **설치 페이지 h1 배지 표시**에만 쓰인다(index.html 이 fetch).

## 계산 규칙 (2026-07-16 사용자 확정)

```
커버 = 기본무료 30분 + Σ(적용된 할인 dc_time)
여유 = 커버 − 경과시간
  여유 >= 0  → 0원. "앞으로 N분 여유" 표시
  여유 <  0  → 부족. 무료2시간(미사용 시)을 먼저 쓰고,
               남은 만큼을 최소 비용 조합(DP)으로 채운다
```

- **기본 무료 30분**은 사용자가 확정해 준 이 주차장 규칙이다. 사이트 데이터에서
  유도한 값이 아니다. 규칙이 바뀌면 `BASE_FREE` 를 고친다.
- **안전하게 올린다** — 추천 조합은 항상 경과시간을 넘겨 덮는다. 딱 맞추지 않는다.
- 사용자가 "앞으로 얼마나 여유 있는지"도 보고 싶어했다. 0원 상태에서도 여유 분을 표시.
- **best() 타이브레이크 (2026-07-22)**: 같은 비용이면 장수가 적은 조합을 고른다.
  분당 단가가 30·60·240분권 모두 동일(16.67원/분)이라 300분 등에서 "30분권 7장"
  같은 지저분한 조합이 나오던 걸, "1시간권 3장+30분권 1장"처럼 큰 권종 우선으로 정리.

## 초과적용 안내 (2026-07-22 사용자 확정)

이미 0원인데 **도구의 최소 비용 조합보다 비싸게** 유료권을 적용한 경우(실수로 넉넉히
누른 경우), 노란 배너로 "N원 절약 가능"을 띄운다. `overApplied(elapsed, counts)` —
무료2시간은 항상 활용한다고 보고 `curPaid`(적용된 유료 합)와 `best(경과−30−120).cost`
를 비교, `curPaid > optCost` 일 때만 뜬다. **도구 추천대로(안전 초과 포함) 적용했으면
안 뜬다** (curPaid == optCost).

**"최종 조합"이 아니라 현재분과의 차이(뭘 빼고/뭘 넣어라)를 보여준다** (2026-07-22
사용자 요청 — 이미 적용된 4시간권을 다시 "적용 필요"로 보여주면 헷갈림). 최적 최종
상태 `target = {무료2시간:1} ∪ best().items` 와 현재 `counts` 를 비교해 `remove`(c>t)·
`add`(t>c) 를 낸다. 카드는 3종: `card(t,cnt,'applied'|'add'|'remove')`, remove 는
빨강 "❌ 빼세요". `hasAdd` 없으면 "빼기만" 안내(순수 제거), 있으면 "빼고 대신 넣기"(스왑).
- 실측 ①: 경과 382분 무료2h+4시간권+1시간권(5,000원) → **1시간권만 빼면** 4,000원 0원 유지(순수 제거).
- 실측 ②: 경과 337분 무료2h+4시간권(4,000원) → 4시간권 빼고 1시간권×3+30분권×1(3,500원) 넣기(스왑).
- 실측 ③: 무료 미사용 30분권×3 → 30분권 빼고 무료2시간 넣기.

모바일 도크는 "💡 -N원 가능" + 빼기/넣기 칩으로 압축. **도구는 자동 제거/적용을 하지
않는다 — 사람에게 안내만** 하므로 "주차권을 실제로 적용하지 말 것" 절대 규칙과 무관.

## 함정

- **이 관리자 화면은 요금(원)을 계산하지 않는다.** `finalFee` 는 항상 `""`,
  `parkEntry.dFee` 는 항상 `0`이다 — 20시간 주차한 무할인 차량도 그렇다.
  요금은 출차 시점에 계산되는 듯하다. **`finalFee`/`dFee` 에 기대지 마라.**
  그래서 이 도구는 전부 **시간**으로 계산한다.
- **`grace_time` 은 쓰지 마라.** 값이 -1, 181, 888, 1290, 112336 으로 들쭉날쭉하고
  `incar_min` 과 상관관계가 없다. 의미를 특정하지 못했다. 무시하기로 했다.
- **할인내역 표는 `<table>` 이 아니다.** dhtmlx 그리드(`XgridDtl`)라 DOM 에서
  `<table>` 로 긁으면 헤더만 나온다. 적용된 할인은 `window.dataSetDtl` 에서 읽어야 한다.
- **`javascript_tool` 로 조사할 때 응답 원문을 그대로 반환하면 차단된다**
  ("Cookie/query string data"). 입차이미지 URL 의 쿼리스트링 때문으로 보인다.
  구조·키만 뽑거나 숫자를 마스킹해서 반환하면 통과한다.
- **모바일 북마크는 긴 javascript: URL 을 조용히 무시한다.** 에러 없이 아무 일도
  안 일어난다. 짧은 alert 테스트로 실행 경로부터 확인할 것. → 로더 방식의 이유.
- **GitHub Pages 가 push 를 받고도 빌드를 누락한 사례가 실제로 있었다**
  (머지 push 에 pages-build-deployment 런이 아예 안 생김). 배포가 이상하면
  Actions 에서 최신 런이 어느 커밋인지 확인하고, 아무 커밋이나 push 해서
  재트리거하면 된다. 반영 후에도 CDN 캐시 최대 10분.
- **file:// 로 index.html 을 브라우저 자동화로 못 연다.** 사용자가 직접 더블클릭하면 된다.
- 북마클릿은 다시 누르면 닫히는 토글이다. 패널 갱신은 `#peId` 를 1초마다 폴링한다
  (사이트 함수를 감싸지 않는다 — 사이트 코드에 손대지 않는 게 안전하다).

## UI 동작 (2026-07-16 개편, 사용자 확정)

- 추천 주차권은 **카드 리스트**. **현재 실제로 적용돼 있는 할인 전부**(`counts`)를
  "현재 적용된 할인" ✓카드로 보여주고, 남은 부족분은 매 폴링마다 **재계산** — 추천과
  다른 권종을 적용해도 맞춰 간다 (특정 권종이 소진돼 대체 사용하는 경우가 실제로 있음).
- **2026-07-22 버그 수정**: 예전엔 PC 가 "패널 연 뒤 새로 적용된 것(현재−base 스냅샷)"만
  ✓카드로 그려서, 적용 도중 **새로고침하면** base 가 기존 적용분을 포함해 카드가 사라졌다.
  이제 PC·모바일 모두 `counts` 전체를 보여준다 (base/newly 스냅샷 로직 제거).
- 완료/부족/대기는 **헤더 색 + 배지**(초록 ✓ 0원 완료 / 빨강 ⚠ 적용 필요 / 회색 대기).
- 0원 상태는 "N시간 N분 남음 · **HH:MM까지** 0원" 을 크게 표시.
  `incar_min` 은 재조회 전까지 고정이므로 값이 갱신된 시각(seenAt)을 기억해
  현재 경과를 보간, 1초마다 실시간 카운트다운한다.

## 조사 방법 메모

차량번호는 **2자리 이상**이면 검색된다. 입차 중인 차를 모를 땐 흔한 조합("23" 등)을
넣으면 수십 대가 잡힌다. 그중 `dscnt_cnt > 0` 인 차를 선택하면 `dataSetDtl` 에
적용된 할인이 들어와, **주차권을 적용해 보지 않고도** 표시 방식을 확인할 수 있다.
